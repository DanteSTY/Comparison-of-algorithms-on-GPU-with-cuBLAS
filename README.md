# Реализование и сравнение производительности различных методов перемножения квадратных матриц C = A * B на графическом процессоре (GPU) с использованием технологии NVIDIA CUDA.


**1)Наивный алгоритм (Naive Implementation)**
Каждый поток (thread) на GPU вычисляет один элемент результирующей матрицы Cij
Для этого поток считывает всю i-ю строку матрицы A и весь j-й столбец матрицы B напрямую из Глобальной памяти (Global Memory) видеокарты.


**2) Блочный алгоритм (Tiled Implementation using Shared Memory)**
- Используется оптимизация работы с памятью. Вычисления разбиваются на блоки (тайлы) размером  TILE_SIZE x TILE_SIZE.
- Потоки блока совместно (кооперативно) загружают фрагменты матриц A и B в Разделяемую память (Shared Memory). Разделяемая память — это быстрый программируемый кэш внутри мультипроцессора.
- Барьерная синхронизация (__syncthreads()) гарантирует, что все данные загружены.
- Вычисления производятся с данными из быстрой разделяемой памяти.
- Процесс повторяется для следующих фрагментов.
Преимущество: Данные считываются из медленной глобальной памяти один раз, а используются многократно (столько раз, каков размер блока). Это значительно снижает нагрузку на шину памяти.
